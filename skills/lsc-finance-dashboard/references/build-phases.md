# Build Phases

- product spec
- metric dictionary
- data ontology
- source maps
- build phases

## Phase 1

Implement the core schema:

- companies
- race events
- sponsors or customers
- contracts
- revenue records
- invoices
- payments
- expenses
- commercial targets
- import metadata

## Phase 2

Implement the import pipeline:

- raw tables
- mapping logic
- normalization rules
- lineage preservation

## Phase 3

Implement analytics views:

- consolidated overview metrics
- monthly finance summary
- receivables aging
- payments due
- TBR race P&L
- sponsor summary
- commercial target progress

## Phase 4

Implement application services and pages:

- Overview
- TBR
- Costs
- Payments
- Commercial Goals
- AI Analysis

## Phase 5

Validate:

- metric correctness
- data lineage
- page behavior
- edge cases
- regression coverage
